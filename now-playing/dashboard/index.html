Set Up your Dashboard

This step by step guide will help you set up your Spotify Now Playing Dashboard.

1. Create a Spotify Developer Account

Go to the Spotify Developer Site and create an account if you don't have one already.

2. Go to the Spotify Developer Dashboard

Click on your name and select "Dashboard" from the dropdown menu.

3. Create a New Application

Click on "Create App" to create a new Spotify Application (Note: Due to changes, you can only create one application per account and your account must be a paid subscriber)

4. Configure Redirect URI

Set the Redirect URI to https://projects.itsmarian.is-a.dev/now-playing/token

5. Client ID and Client Secret

Open https://projects.itsmarian.is-a.dev/now-playing/token

Copy the Client ID and paste it into the "Client ID" field.

Then, click on "View Client Secret" and copy the secret, then paste it into the "Client Secret" field.

6. Select scopes and get token

Select "user-read-recently-played", "user-read-currently-playing" and "user-read-playback-state" in the scopes section, and then click "Get Spotify Access Code".

After that you will be redirected to Spotify, where you need to approve the permissions.

7. Refresh Token

Then, you will be redirected to this page with a refresh token. Copy this token.

Then navigate to https://projects.itsmarian.is-a.dev/now-playing/dashboard

8. Dashboard

On the dashboard, you can see three fields, your Client ID and Client Secret will be filled in automatically.

9. Paste Refresh Token

If the token is not already in the field, paste the refresh token you copied into the "Refresh Token" field on the dashboard.

API Reference

The Now Playing API allows you to retrieve information about the song currently playing on Spotify.

Endpoint

GET https://itsmarianmc-github.vercel.app/api/now-playing

Query Parameter

Parameter Typ Beschreibung
client_idRequired String Your Spotify Application Client ID
client_secretRequired String Your Spotify Application Client Secret
refresh_tokenRequired String Your Spotify Refresh Token

Rate Limits

For the Demo Version there is no rate limit, but for the production version, there is a rate limit of 4 seconds per request. Due to limitations there I can't set a higher rate limit.

Main fields

Field Type Description
playing Boolean Indicates whether music is currently playing
timestamp Number Unix timestamp of the query in milliseconds
progress_ms Number Current progress of the track in milliseconds
track Object Information about the current track (id, name, duration_ms, uri, spotify_url, etc.)
track.duration_ms Number Total length of the track in milliseconds
artists Array List of artists with id, name, uri, and spotify_url
album Object Album information including cover images, release_date, and total_tracks
album.images Array Cover images in various sizes (640x640, 300x300, 64x64)
progress_percent String Progress in percent as a formatted string
remaining_ms Number Remaining time of the track in milliseconds
estimated_end_time Number Estimated Unix timestamp (ms) when the track ends

API v1


API Endpoint

GET https://projects.itsmarian.is-a.dev/now-playing/v1/

This is the first version of the now playing widget.

This version, displays the current song in a modern animation to save screen space. It appears 10 seconds after the start of the song, the middle of the song and 20 seconds before the end of the song.

Open API Endpoint

API v2


API Endpoint

GET https://projects.itsmarian.is-a.dev/now-playing/v2/

This is the second version of the now playing widget.

This version is almost the same as the first version. It displays the current song in a modern and minimalistic animation to save screen space. It also appears 10 seconds after the start of the song, the middle of the song and 20 seconds before the end of the song.

However, in this version the animation transitions for a few seconds to the album art and then back to original.

API v3


API Endpoint

GET https://projects.itsmarian.is-a.dev/now-playing/v3/

This is the third version of the now playing widget.

This version constantly displays the album art, title and artist information as well as the progress bar and remaining time.

It is the most feature-rich version of the now playing widget and it adapts its scale individually to the size of the screen.
